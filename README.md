# Quantum Risk Index: Supplementary Materials

Reproducibility package for the paper

> **A Quantum Risk Index for Cryptographic CVEs: Empirical Evidence
> from the National Vulnerability Database, 2016--2026.**
> *Submitted to MDPI Cryptography / MDPI Future Internet, 2026.*

The repository contains all the data, code and figure-generation scripts
needed to reproduce every numerical claim, table and figure in the
manuscript from the raw NVD JSON feeds and the CISA Known Exploited
Vulnerabilities (KEV) catalogue.

## Quick start

```bash
# 1. Clone the repository.
git clone https://github.com/<user>/quantum-risk-index.git
cd quantum-risk-index

# 2. (Recommended) Create a clean Python environment.
python -m venv .venv
source .venv/bin/activate           # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# 3. Provide the input data.
#    - Place the NVD bulk feeds (nvdcve-2.0-YYYY.json, 2016--2026) and
#      known_exploited_vulnerabilities.csv into raw/.
#    - Optional (only for epss_and_validation.py): download an EPSS bulk
#      feed from https://first.org/epss/data_stats and place the
#      resulting epss_scores-YYYY-MM-DD.csv.gz file into raw/.

# 4. Build the corpus (one-off; produces raw/hndl_dataset.csv, ~78k rows).
python supplementary/parse_all_nvd_hndl.py

# 5. Reproduce the statistical tests, sensitivity analysis and figures.
python supplementary/step3_hndl_tests.py
python supplementary/sensitivity_analysis.py
python supplementary/epss_and_validation.py
python supplementary/make_figures.py
```

The full pipeline runs in roughly two minutes on a contemporary laptop.
A pre-built `raw/hndl_dataset.csv` is also shipped with the repository so
that the downstream steps (4 onwards) can be executed without first
re-running the NVD parser.

## Script inventory

| Script | Reproduces |
|--------|------------|
| `parse_all_nvd_hndl.py`        | Sections 3.2--3.5; produces `raw/hndl_dataset.csv` |
| `recompute_qri_v2.py`          | Section 3.5 (audit utility for the QRI formula only) |
| `step3_hndl_tests.py`          | Section 4.1--4.4 (Q1--Q4 hypothesis tests, Tables 4--6) |
| `sensitivity_analysis.py`      | Section 4.5; Tables 7--8 and the QRI vs CVSS comparison |
| `epss_and_validation.py`       | Section 4.5.3 (EPSS, Table 9) and Section 3.3.1 (convergent validity) |
| `make_figures.py`              | Figures 1--8 |

All scripts use relative paths (`raw/`, `results/`, `figures/`) computed
from the script's own location, so they run unchanged on any operating
system.

## Reproducing the headline numbers

Running `step3_hndl_tests.py` against the supplied `hndl_dataset.csv`
reproduces, among others:

* Mann--Whitney U comparing KEV vs non-KEV QRI:
  U = 24,320,998, *p* = 2.57 x 10^-63, +27.2 % mean separation.
* chi^2 for Shor-vulnerable CVEs in CISA KEV:
  chi^2(1) = 8.286, *p* = 0.004.
* chi^2 for the HS-tier partition (HS = 0, HS = 1--3, HS >= 4):
  chi^2(2) = 27.812, *p* < 0.001.
* Annual growth of the crypto-relevant CVE corpus (2016--2026 inclusive):
  beta_1 = 0.105, R^2 = 0.447, CAGR = 11.0 %, *p* = 0.025.
* Annual growth of Shor-vulnerable CVEs:
  beta_1 = 0.084, R^2 = 0.420, CAGR = 8.8 %, *p* = 0.031.
